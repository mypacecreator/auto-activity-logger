# auto-activity-logger

ChatworkとGitHubから「自分の行動ログだけ」を自動抽出し、Markdown形式で保存するCLIツール。
チャット画面やGitHubのUIを開かずに、通知ノイズを受けることなく実績だけを記録できます。

---

## 特徴

- **Chatwork**: 指定ルーム（または全参加ルーム）から自分の発言のみを抽出
- **GitHub**: Push / PR / Issue / Comment など主要イベントを時系列で記録
- **スクリーンショット連携**: CleanShot X のスクショをタイムスタンプで活動ログと突合し、文脈を推測
- **ルーム自動絞り込み**: `last_update_time` を使い、対象日に更新のないルームをスキップ（API コール削減）
- **NotebookLM / Gemini 対応構造**: 生成したMarkdownをAIに読み込ませて振り返りに活用できる形式

## 認証情報の取得

### Chatwork API トークン

1. [Chatwork API トークン発行ページ](https://www.chatwork.com/service/packages/chatwork/subpackages/api/token.php) でトークンを発行
2. 自分の `account_id` は以下で確認：
   ```bash
   curl -H "X-ChatWorkToken: <YOUR_TOKEN>" https://api.chatwork.com/v2/me
   ```
   レスポンスの `account_id` フィールドをメモしておく

### GitHub Personal Access Token (PAT)

1. [GitHub Settings > Tokens](https://github.com/settings/tokens) で PAT を作成
2. 必要なスコープ：
   - `read:user` — パブリックイベントの取得
   - `repo` — プライベートリポジトリのイベントも取得する場合

## 環境変数一覧

`.env` ファイルに以下の変数を設定します。

| 変数名 | 必須 | デフォルト | 説明 |
|---|:---:|---|---|
| `CHATWORK_API_TOKEN` | ◎ | — | Chatwork API トークン |
| `CHATWORK_MY_ACCOUNT_ID` | ◎ | — | 自分のアカウントID（発言フィルタ用） |
| `CHATWORK_ROOM_IDS` | | 空（全ルーム） | 対象ルームID（カンマ区切り: `12345,67890`）。空の場合は全参加ルームを走査 |
| `GITHUB_TOKEN` | ◎ | — | GitHub Personal Access Token |
| `GITHUB_USERNAME` | ◎ | — | GitHub ユーザー名 |
| `DEFAULT_DATE` | | `yesterday` | デフォルトの取得日。`today` または `yesterday` |
| `OUTPUT_DIR` | | `logs` | Markdown の出力先ディレクトリ（プロジェクトルートからの相対パス） |
| `SCREENSHOT_DIR` | | `~/Downloads/CleanShot` | CleanShot X のスクリーンショット保存先 |

### 設定例

```dotenv
CHATWORK_API_TOKEN=abc123xyz
CHATWORK_MY_ACCOUNT_ID=123456
CHATWORK_ROOM_IDS=111111,222222   # 空にすると全ルームを自動走査

GITHUB_TOKEN=ghp_xxxxxxxxxxxx
GITHUB_USERNAME=your-username

DEFAULT_DATE=yesterday
OUTPUT_DIR=logs
SCREENSHOT_DIR=~/Downloads/CleanShot
```

## 使い方

```bash
# 前日分のログを生成（デフォルト動作）
npm start

# 当日分のログを生成
npm start -- --date today

# 前日分・スクリーンショットなし
npm start -- --date yesterday --no-screenshots

# GitHub のみ（Chatwork をスキップ）
npm start -- --no-chatwork

# Chatwork のみ（GitHub をスキップ）
npm start -- --no-github

# スクリーンショットのみ確認
npm start -- --no-chatwork --no-github
```

### オプション一覧

| オプション | 説明 |
|---|---|
| `--date <today\|yesterday>` | 取得する日付。未指定時は `DEFAULT_DATE` の値を使用 |
| `--no-chatwork` | Chatwork の取得をスキップ |
| `--no-github` | GitHub の取得をスキップ |
| `--no-screenshots` | スクリーンショットスキャンをスキップ |

## 出力ファイル

`logs/YYYY-MM-DD_activity.md` として保存されます。

### サンプル

```markdown
# Activity Log — 2026-03-04

> Generated by auto-activity-logger on 2026-03-05 09:00:00 UTC

---

## [10:30] 💬 Chatwork (案件Aチャンネル)

- **Message**: ここのマージン、あと5px詰めてください…

## [10:32] 📸 Screenshot & 💬 Chatwork (案件Aチャンネル)

- **Screenshot:** CleanShot 2026-03-04 at 10.32.15@2x.png
- **Chatwork (案件Aチャンネル):** ここのマージン、あと5px詰めて…
- **推測:** 指示出しまたはメッセージの参照資料

## [14:15] 🐙 GitHub (owner/repo)

- **PushEvent**: Push: fix: ログ出力のフォーマット修正 (+2 more)

---

## Summary

- Chatwork messages: **3**
- GitHub events: **5**
- Screenshots: **2** (2 matched to activities)
- Total activities: **10**

### GitHub repositories
- owner/repo-a
- owner/repo-b

### Chatwork rooms
- 案件Aチャンネル
- 社内雑談
```

## GitHub 対応イベント

| イベント | ログに出力される内容 |
|---|---|
| `PushEvent` | コミットメッセージ（複数件の場合は件数も表示） |
| `PullRequestEvent` | PR タイトル + アクション（opened / closed / merged など） |
| `PullRequestReviewEvent` | レビューアクション + PR タイトル |
| `PullRequestReviewCommentEvent` | レビューコメント冒頭 80 文字 |
| `IssuesEvent` | Issue タイトル + アクション |
| `IssueCommentEvent` | コメント先の Issue タイトル + コメント冒頭 |
| `CreateEvent` | ブランチ / タグの作成 |
| `DeleteEvent` | ブランチ / タグの削除 |
| `ForkEvent` | フォーク先リポジトリ名 |
| `ReleaseEvent` | リリース名 / タグ名 |

> GitHub API の仕様上、取得できるイベントは最新 300 件（60件 × 5ページ）までです。

## スクリーンショット機能

**CleanShot X** で撮影したスクリーンショットを活動ログと自動で関連付けます。

### 仕組み

1. `SCREENSHOT_DIR` 内の `.png` / `.jpg` ファイルをスキャン
2. ファイル名からタイムスタンプを抽出（対応パターン: `CleanShot YYYY-MM-DD at HH.mm.ss`）
3. 対象日のファイルのみを処理
4. 各スクリーンショットの前後 **±5分以内** にある ActivityEntry と突合
5. 最も近いアクティビティの種別に応じて文脈を推測：

| 突合結果 | 推測テキスト |
|---|---|
| Chatwork メッセージ | 指示出しまたはメッセージの参照資料 |
| GitHub イベント | 実装中の不具合または参照の備忘録 |
| 突合なし | （推測なし・ファイル名のみ記録） |

### ファイル名パターン

CleanShot X のデフォルト保存形式に対応しています：

```
CleanShot 2026-03-04 at 23.30.00@2x.png
CleanShot 2026-03-04 at 23.30.00.png
```
